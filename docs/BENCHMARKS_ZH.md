# 推理性能基准

## Windows 11 + RTX 3080 10GB (CUDA)

7 张猫图，各 3 轮，`max_new_tokens=512`，自动长边 800px。

| 模式 | 图像尺寸 | 冷启动 (R1) | 各轮详情 | 稳定平均 (热启动) | 显存/内存开销 |
|------|---------|-----------|--------|-----------------|-------------|
| VLM only | 大图 (800×640) | ~22.0s* | 1.0s (平均耗时) | **1.0s/张** | ~7.5GB+ (峰值) |
| VLM only | 缩略图 | 367ms | 367ms (平均耗时) | **367ms/张** | ~5.5GB (加载后) |
| VLM + SAM2 | 大图 (800×640) | 3.4s | 3.4s (平均耗时) | **3.4s/张** | ~7.5GB+ (峰值) |
| VLM + SAM2 | 缩略图 | 475ms | 475ms (平均耗时) | **475ms/张** | ~5.5GB (加载后) |

> *冷启动含第 1 轮第 1 张模型加载时间（~22s）。
> **注**：10GB 显存可能触顶导致降速，推荐使用 12GB+ 显卡。

## macOS Apple Silicon 24GB (MPS)

13 张猫图，各 3 轮，`max_new_tokens=512`，自动长边 1024px。

| 模式 | 图像尺寸 | 冷启动 (R1) | 各轮详情 (R2 / R3) | 稳定平均 (热启动) | 显存/内存开销 |
|------|---------|-----------|------------------|-----------------|-------------|
| VLM only | 默认 (1024px) | 3.7s | 4.3s / 4.3s | **4.3s/张** | 9.8–13GB 稳定 |
| VLM + SAM2 | 默认 (1024px) | 13.8s* | 4.9s / 4.9s | **4.9s/张** | 9.8–13GB 稳定 |

> *冷启动包含 VLM + SAM2 模型加载（~14s）。SAM2 额外开销：+0.65s (15%)。Mask 成功率：13/13。
> **注**：6 轮全程 9.8–13GB 稳定，每次检测后 MPS 清理策略生效。

## Linux + RTX 4080 16GB (CUDA)

13 张猫图，各 4 轮，自动长边 1024px (大图) / 256px (缩略图)。

| 模式 | 图像尺寸 | 冷启动 (R1首张) | 各轮详情 (R2/R3/R4) | 稳定平均 (热启动) | 显存/内存开销 |
|------|---------|-----------|------------------|-----------------|-------------|
| VLM only | 大图 (1024px) | 2.14s | 2.76s / 2.75s / 2.75s | **2.75s/张** | ~7.7GB (峰值) |
| VLM only | 缩略图 (256px) | 12.37s* | 0.64s / 0.59s / 0.58s | **0.60s/张** | ~7.7GB (峰值) |
| VLM + SAM2 | 大图 (1024px) | 2.62s | 3.69s / 3.74s / 3.69s | **3.71s/张** | ~8.2GB (峰值) |
| VLM + SAM2 | 缩略图 (256px) | 2.49s | 0.96s / 0.92s / 0.94s | **0.94s/张** | ~8.2GB (峰值) |

> *完全冷启动 (12.37s) 包含模型权重首次载入显存与 PyTorch 计算图的初次构建。其余“首张”为同进程下模型预热后的切换耗时。
> **注**：在高频大图推理中，VLM+SAM2 平稳后的理论耗时极度稳定，基本平滑锁定在 3.71s。

## 配置说明

- **长边上限**：根据 GPU 显存自动选择（<12GB → 800px，12–16GB → 1024px，≥16GB → 1333px；MPS → 1024px）
- **GPU 内存管理**：策略模式（`gpu_memory.py`）— CUDA `expandable_segments:True` / MPS `synchronize`+`empty_cache`+`gc`
- **SAM2 模型**：`facebook/sam2.1-hiera-base-plus`
