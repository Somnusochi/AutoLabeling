# 训练参数调优

合理的训练参数是获得高性能 YOLO 模型的关键。本指南帮助你根据任务需求选择最佳参数。

## YOLO 系列选择

### 各版本对比

| 版本 | 特点 | 推荐场景 |
|------|------|---------|
| **YOLOv5** | 经典稳定，生态成熟 | 需要兼容旧系统，或追求稳定性 |
| **YOLOv8** | 速度与精度平衡，官方推荐 | 通用场景，大多数任务的首选 |
| **YOLOv11** | 最新架构，性能最佳 | 追求最高精度，不介意训练时间 |
| **YOLOv26** | 实验性，最前沿技术 | 研究和探索，生产环境慎用 |

**推荐**：新手使用 **YOLOv8**，追求性能使用 **YOLOv11**。

### 模型规格选择

| 规格 | 参数量 | 速度 | 精度 | 显存需求 | 推荐场景 |
|------|--------|------|------|---------|---------|
| **n** (nano) | 最小 | 最快 | 较低 | 2GB+ | 边缘设备、实时检测 |
| **s** (small) | 小 | 快 | 中等 | 4GB+ | 移动端、嵌入式 |
| **m** (medium) | 中等 | 中等 | 较高 | 6GB+ | 通用推荐，平衡速度与精度 |
| **l** (large) | 大 | 较慢 | 高 | 8GB+ | 高精度需求 |
| **x** (xlarge) | 最大 | 最慢 | 最高 | 12GB+ | 追求极限精度 |

**推荐**：
- **8GB 显存**：使用 `m` 或 `s`
- **12GB+ 显存**：使用 `l` 或 `x`
- **边缘部署**：使用 `n` 或 `s`

## 核心训练参数

### Epochs（训练轮数）

训练轮数决定模型学习数据集的次数。

| 数据集大小 | 推荐 Epochs | 说明 |
|-----------|-------------|------|
| **< 100 张** | 300-500 | 小数据集需要更多轮数 |
| **100-300 张** | 200-300 | 中等数据集，标准配置 |
| **300-500 张** | 150-200 | 较大数据集，适当减少 |
| **> 500 张** | 100-150 | 大数据集，避免过拟合 |

**如何判断是否过拟合**：
- 训练 Loss 持续下降，但验证 Loss 开始上升
- 训练集 mAP 高，但测试集 mAP 低
- 模型在训练图片上表现完美，但在新图片上表现差

**解决方法**：
- 减少 Epochs
- 增加数据量
- 使用数据增强

### ImgSz（输入图像尺寸）

输入图像尺寸影响检测精度和速度。

| 尺寸 | 速度 | 精度 | 推荐场景 |
|------|------|------|---------|
| **416** | 最快 | 较低 | 小目标少，追求速度 |
| **512** | 快 | 中等 | 平衡速度与精度 |
| **640** | 中等 | 较高 | **通用推荐**，大多数场景 |
| **768** | 较慢 | 高 | 小目标多，需要高精度 |
| **1024** | 最慢 | 最高 | 极端精度需求 |

**推荐**：使用 **640**，这是 YOLO 的标准输入尺寸。

**特殊情况**：
- 如果目标很小（< 32x32 像素），使用 768 或 1024
- 如果追求实时检测，使用 416 或 512

### Batch（批大小）

批大小影响训练稳定性和显存占用。

| 显存 | 推荐 Batch | 说明 |
|------|-----------|------|
| **4GB** | 4-8 | 小显存，使用小批量 |
| **6GB** | 8-12 | 中等显存 |
| **8GB** | 12-16 | 通用推荐 |
| **12GB+** | 16-32 | 大显存，可以使用大批量 |

**如何确定最大 Batch**：
1. 从 16 开始尝试
2. 如果显存不足（OOM 错误），减半到 8
3. 如果还有余量，增加到 24 或 32

**注意**：Batch 越大，训练越稳定，但显存占用也越大。

## 高级参数调优

### 学习率（Learning Rate）

系统自动调整学习率，但你可以手动设置：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **lr0** | 0.01 | 初始学习率 |
| **lrf** | 0.01 | 最终学习率（相对于 lr0） |

**调参建议**：
- 训练不稳定（Loss 震荡）：降低 lr0 到 0.005
- 训练太慢：提高 lr0 到 0.02
- 一般情况：使用默认值

### 数据增强

系统内置了多种数据增强技术：

| 增强方式 | 效果 | 推荐 |
|---------|------|------|
| **翻转** | 水平/垂直翻转 | ✅ 强烈推荐 |
| **旋转** | 随机旋转 ±15° | ✅ 推荐 |
| **缩放** | 随机缩放 0.5-2.0 倍 | ✅ 推荐 |
| **裁剪** | 随机裁剪 | ✅ 推荐 |
| **颜色抖动** | 调整亮度/对比度/饱和度 | ✅ 推荐 |
| **马赛克** | 4 张图拼接 | ✅ 强烈推荐 |
| **混合** | 2 张图混合 | ⚠️ 可选 |

**建议**：保持默认设置，系统已经优化了数据增强策略。

### 优化器选择

| 优化器 | 特点 | 推荐 |
|--------|------|------|
| **SGD** | 经典优化器，收敛稳定 | 通用推荐 |
| **Adam** | 自适应学习率，收敛快 | 小数据集推荐 |
| **AdamW** | Adam + 权重衰减 | 防止过拟合 |

**推荐**：使用默认的 **SGD**。

## 训练监控

### 关键指标

训练过程中需要关注的指标：

| 指标 | 含义 | 目标值 |
|------|------|--------|
| **Loss** | 损失函数值 | 持续下降 |
| **mAP50** | IoU=0.5 时的平均精度 | > 0.8 为优秀 |
| **mAP50-95** | IoU 从 0.5 到 0.95 的平均精度 | > 0.5 为优秀 |
| **Precision** | 精确率（检测到的目标中有多少是正确的） | > 0.8 |
| **Recall** | 召回率（所有目标中有多少被检测到） | > 0.8 |

### 训练曲线解读

**正常训练**：
```
Loss: ↘️ 持续下降
mAP: ↗️ 持续上升
Train Loss 和 Val Loss 同步下降
```

**过拟合**：
```
Train Loss: ↘️ 持续下降
Val Loss: ↗️ 开始上升（在某个 Epoch 后）
Train mAP 高，Val mAP 低
```

**欠拟合**：
```
Loss: ➡️ 下降缓慢或停滞
mAP: ➡️ 上升缓慢或停滞
Train 和 Val 指标都很低
```

### 常见问题排查

**问题 1：Loss 不下降**
- 原因：学习率太小、数据质量问题、类别标注错误
- 解决：提高学习率、检查数据质量、验证标注正确性

**问题 2：Loss 震荡**
- 原因：学习率太大、Batch 太小、数据不均衡
- 解决：降低学习率、增加 Batch、平衡类别分布

**问题 3：显存不足（OOM）**
- 原因：Batch 太大、ImgSz 太大、模型太大
- 解决：减小 Batch、降低 ImgSz、选择更小的模型规格

**问题 4：训练太慢**
- 原因：Batch 太小、ImgSz 太大、模型太大
- 解决：增加 Batch、降低 ImgSz、选择更小的模型规格

## 模型评估

### 评估指标解读

| 指标 | 含义 | 优秀 | 良好 | 一般 |
|------|------|------|------|------|
| **mAP50** | IoU=0.5 时的平均精度 | > 0.9 | 0.7-0.9 | 0.5-0.7 |
| **mAP50-95** | 严格评估（IoU 从 0.5 到 0.95） | > 0.7 | 0.5-0.7 | 0.3-0.5 |
| **Precision** | 精确率 | > 0.9 | 0.7-0.9 | 0.5-0.7 |
| **Recall** | 召回率 | > 0.9 | 0.7-0.9 | 0.5-0.7 |

### 指标优先级

根据应用场景选择重点优化的指标：

| 场景 | 优先指标 | 说明 |
|------|---------|------|
| **安全监控** | Recall | 宁可误报，不能漏报 |
| **自动驾驶** | Precision + Recall | 两者都要高 |
| **工业检测** | Precision | 避免误判导致停线 |
| **通用检测** | mAP50-95 | 综合评估 |

### 混淆矩阵分析

训练完成后查看混淆矩阵：
- **对角线**：正确分类
- **非对角线**：错误分类
- **某行值高**：该类别容易被误判为其他类别
- **某列值高**：其他类别容易被误判为该类别

**改进方向**：
- 增加容易混淆的类别的样本
- 检查类别标注是否一致
- 考虑合并相似类别

## 模型优化

### 1. 模型蒸馏

使用大模型指导小模型训练：
1. 用 `l` 或 `x` 模型训练
2. 用训练好的大模型标注更多数据
3. 用这些数据训练 `s` 或 `m` 模型

**效果**：小模型获得接近大模型的性能。

### 2. 模型剪枝

减少模型参数，提高推理速度：
```bash
# 使用 ultralytics 剪枝
yolo prune model=best.pt amount=0.3
```

**效果**：模型大小减少 30%，速度提升 20-30%，精度损失 < 1%。

### 3. 模型量化

将 FP32 转换为 INT8，大幅提升速度：
```bash
# 导出 INT8 模型
yolo export model=best.pt format=engine int8=True
```

**效果**：速度提升 2-3 倍，精度损失 1-2%。

### 4. TensorRT 加速

在 NVIDIA GPU 上使用 TensorRT 加速：
```bash
# 导出 TensorRT 模型
yolo export model=best.pt format=engine
```

**效果**：推理速度提升 2-5 倍。

## 实战案例

### 案例 1：人员检测（简单任务）

**数据集**：200 张图片，1 个类别（person）

**推荐配置**：
```
模型：YOLOv8s
Epochs：200
ImgSz：640
Batch：16
```

**预期效果**：mAP50 > 0.9，mAP50-95 > 0.6

### 案例 2：交通场景检测（中等任务）

**数据集**：500 张图片，5 个类别（car, truck, bus, motorcycle, bicycle）

**推荐配置**：
```
模型：YOLOv8m
Epochs：150
ImgSz：640
Batch：12
```

**预期效果**：mAP50 > 0.85，mAP50-95 > 0.5

### 案例 3：工业缺陷检测（复杂任务）

**数据集**：1000 张图片，8 个类别（各种缺陷类型）

**推荐配置**：
```
模型：YOLOv8l
Epochs：300
ImgSz：768（缺陷较小）
Batch：8
```

**预期效果**：mAP50 > 0.8，mAP50-95 > 0.4

## 总结

参数调优的核心原则：
1. **从默认值开始**：系统默认参数已经优化，适合大多数场景
2. **逐步调整**：一次只改一个参数，观察效果
3. **监控指标**：关注 Loss、mAP、Precision、Recall
4. **避免过拟合**：训练集和验证集指标要同步提升
5. **根据场景选择**：不同场景对速度和精度的要求不同

**快速开始配置**：
```
模型：YOLOv8m
Epochs：200
ImgSz：640
Batch：16
```

如果效果不好，参考本指南逐步调优。
